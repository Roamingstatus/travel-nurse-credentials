Update the existing Credanta app.

Add a lightweight Analytics panel inside the existing Admin panel.

Do NOT rebuild the app.

Do NOT modify unrelated app functionality.

────────────────────────
ADMIN ANALYTICS PANEL
────────────────────────

Add a new section/page in the admin panel:

Title:
“Analytics”

Subtitle:
“Lightweight product usage tracking for Credanta.”

If admin navigation exists, add:
Analytics

/admin/analytics

Protect this route so only admin users can access it.

────────────────────────
EVENT DATABASE
────────────────────────

Create or reuse an events table.

Fields:

- id
- user_id
- event_type
- event_data
- created_at

event_type examples:
- document_uploaded
- document_previewed
- packet_generated
- premium_clicked
- reminders_enabled
- checklist_used

event_data should be JSON if supported.

Example event_data:
{
  "document_id": "123",
  "category": "Certification",
  "source": "credentials_page"
}

────────────────────────
EVENT LOGGING HELPER
────────────────────────

Create reusable helper:

log_event(user_id, event_type, event_data=None)

Requirements:
- should never crash the user-facing app
- if logging fails, write warning to server logs only
- do not expose technical errors to user
- keep implementation lightweight

Use this helper throughout the app.

────────────────────────
WHERE TO LOG EVENTS
────────────────────────

Log document_uploaded when:
- user successfully uploads a document

Log document_previewed when:
- user opens preview modal
- user hovers preview if hover preview exists
- avoid duplicate spam by throttling hover previews if possible

Log packet_generated when:
- user successfully generates PDF/ZIP packet

Log premium_clicked when:
- user clicks upgrade button
- user clicks locked premium feature
- user clicks Premium or Premium+ feature card

Log reminders_enabled when:
- user enables email reminders
- user enables SMS reminders
- user saves reminder settings as enabled

Log checklist_used when:
- user generates smart checklist
- user views checklist results
