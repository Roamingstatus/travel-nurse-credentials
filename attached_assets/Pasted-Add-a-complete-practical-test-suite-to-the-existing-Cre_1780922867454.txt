Add a complete practical test suite to the existing Credanta app.

Do NOT rebuild the app.
Do NOT change working features unless tests expose a bug.
Do NOT add unnecessary tools.

Goal:
Create tests beyond basic debugging, security checks, and speed tests.

Focus on:
- real user workflows
- feature reliability
- permissions
- data privacy
- file handling
- premium gating
- reminders
- mobile/browser behavior

────────────────────────
1. UNIT TESTS
────────────────────────

Add unit tests for:

- expiration status logic
- custom expiration rules
- NIH/NIHSS 1-year rule
- document category detection
- readiness signal logging
- resume enhancer scoring
- weak word detection
- keyword matching
- reminder date calculations

Example:
- expired document returns "expired"
- document expiring in 7 days returns "expiring_soon"
- NIHSS document with issue date gets expiration date + 1 year

────────────────────────
4. PREMIUM PERMISSION TESTS
────────────────────────

Test subscription tiers.

Free user:
- can upload documents
- can use free resume enhancer
- cannot use premium packet generation if gated
- cannot enable email reminders
- cannot enable SMS reminders
- cannot create Premium+ recruiter links if gated

Premium user:
- can use email reminders
- can use packet generation
- cannot use SMS reminders
- cannot use Premium+ features

Premium+ user:
- can use email reminders
- can use SMS reminders
- can create recruiter share links
- can use smart checklist
- can use agency packet tools

Backend must enforce this.
Do not rely only on frontend hiding.

────────────────────────
5. DATA PRIVACY / MULTI-USER TESTS
────────────────────────

Create tests with two users:

User A:
- uploads document A

User B:
- uploads document B

Verify:
- User A cannot fetch User B document
- User A cannot preview User B file
- User A cannot delete User B document
- User A cannot include User B document in packet
- User A cannot create share link for User B document

This is critical.

- PDF
- JPG

- TXT resume

Test rejected files:

- EXE
- JS
- HTML

- corrupted file

Verify:
- accepted files upload
- rejected files show clear error
- app does not crash

────────────────────────
7. REMINDER TESTS
────────────────────────

Test reminder logic.

Documents expiring in:
- 30 days
- 14 days
- 7 days
- 0 days

Should trigger reminder eligibility.

Documents expiring in:
- 29 days
- 13 days
- 6 days
- -1 days

Should not trigger duplicate scheduled reminder unless intended.

Test:
- duplicate reminders are not sent twice
- missing Resend key does not crash
- missing Twilio key does not crash
- free users cannot receive automated reminders
- premium users receive email only
- premium+ users receive email and SMS if enabled
