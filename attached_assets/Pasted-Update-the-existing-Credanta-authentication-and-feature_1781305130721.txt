────────────────────────
5. BRUTE-FORCE PROTECTION
────────────────────────

Add rate limits:

Login:
- 5 failed attempts per email per 15 minutes
- 10 failed attempts per IP per 15 minutes

Registration:
- 5 attempts per IP per hour

Forgot password:
- 3 attempts per email per hour
- 5 attempts per IP per hour

If exceeded:
“Too many attempts. Please try again later.”

Add optional account cooldown:
- if 10 failed login attempts for same email in 1 hour, temporarily lock login for 15 minutes

Do not reveal whether email exists.

Use same message:
“Invalid email or password.”

────────────────────────
6. XSS / INJECTION HARDENING
────────────────────────

Sanitize and validate all auth inputs.

Email:
- valid email format
- max 254 characters
- lowercase
- trim whitespace

Name:
- max 80 characters
- strip HTML
- reject script tags

Password:
- do not sanitize destructively
- do not log
- only validate length/rules

Use parameterized queries only.

Add security headers if not present:
- Content-Security-Policy
- X-Content-Type-Options: nosniff
- Referrer-Policy
- Permissions-Policy
- X-Frame-Options or frame-ancestors in CSP

Do not display raw user input without escaping.

────────────────────────
7. CSV INJECTION PROTECTION
────────────────────────

For any CSV exports:
- waitlist export
- user export
- beta feedback export
- admin analytics export
- security events export

Sanitize cells starting with:
=, +, -, @, tab, carriage return

Prefix dangerous CSV values with apostrophe.

Example:
=IMPORTXML(...)
becomes:
'=IMPORTXML(...)

Do not allow formulas to execute when exported.

- malformed headers

- csrf_failed
