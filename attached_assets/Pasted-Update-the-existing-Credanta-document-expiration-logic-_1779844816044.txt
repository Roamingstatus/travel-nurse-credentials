Update the existing Credanta document expiration logic.

Goal:
Add a universal custom expiration rule for NIH Stroke Scale / NIHSS documents.

Do NOT rewrite the upload system.
Do NOT rewrite the full app.
Only update expiration detection/rule handling.

Requirement:
Some documents do not declare an expiration date on the file.
For these documents, the app should apply custom expiration rules based on document type.

Add this custom rule:

Document type:
- NIH
- NIHSS
- NIH Stroke Scale
- National Institutes of Health Stroke Scale

Expiration rule:
- Valid for 1 year from the issue date, completion date, certificate date, upload date, or detected document date.
- Prefer detected issue/completion date if available.
- If no date is detected, use upload date as fallback.
- Auto-set expiration_date = base_date + 1 year.

Implementation details:
1. Create or update a centralized expiration rules file/service.
   Suggested name:
   - expiration_rules.py
   or
   - services/expiration_rules.py

2. Add a rules dictionary/list similar to:

CUSTOM_EXPIRATION_RULES = [
  {
    "name": "NIH Stroke Scale",
    "keywords": ["nih", "nihss", "nih stroke scale", "national institutes of health stroke scale"],
    "duration": {"years": 1},
    "base_date_priority": ["issue_date", "completion_date", "certificate_date", "detected_date", "upload_date"]
  }
]

3. When a document is uploaded or parsed:
   - check filename
   - check extracted text
   - check detected document category/title
   - if NIH/NIHSS keyword match is found:
       apply the 1-year expiration rule
       set document.expiration_date
       set document.expiration_rule_applied = "NIH Stroke Scale - 1 year"
       set document.expiration_source = "custom_rule"

4. Add fields if they do not exist:
   - expiration_rule_applied
   - expiration_source

5. If fields cannot be added easily, store this info in document metadata.

6. Update UI document card/details page to show:
   “Expiration set by rule: NIH Stroke Scale - 1 year”
   only when a custom rule was applied.

7. Add tests for:
   - filename contains NIHSS and issue_date exists → expiration = issue_date + 1 year
   - extracted text contains NIH Stroke Scale and completion_date exists → expiration = completion_date + 1 year
   - no detected date but upload_date exists → expiration = upload_date + 1 year
   - unrelated document does not apply NIH rule

8. Make the rule system extensible so more custom rules can be added later for:
   - BLS
   - ACLS
   - PALS
   - TB
   - Fit Test
   - Driver license
   - Professional licenses

Important:
- Avoid hardcoding NIH logic directly inside upload route.
- Create reusable helper:
  apply_custom_expiration_rules(document, extracted_text)
- Keep implementation clean and easy to expand.
- Run tests after implementation.
