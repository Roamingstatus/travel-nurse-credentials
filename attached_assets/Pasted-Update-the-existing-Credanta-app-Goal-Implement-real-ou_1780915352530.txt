EMAIL REMINDERS
────────────────────────

Create email service:

Function:
send_expiration_email(user, document, days_until_expiration)

Email subject examples:

30 days:
Credanta Reminder: Document expires in 30 days

14 days:
Credanta Reminder: Document expires in 14 days

7 days:
Credanta Reminder: Document expires in 7 days

0 days:
Credanta Reminder: Document expires today

Email body:
- user first name if available
- document name
- category
- expiration date
- days remaining
- link to Credanta dashboard
- short reminder to update or renew the document

Do not include private document file URLs in email.

────────────────────────
BACKGROUND SCHEDULER
────────────────────────

Use APScheduler.

Create scheduler service:

services/reminder_scheduler.py

Run once daily.

Suggested schedule:
8:00 AM server time

Job:
check_expiring_documents()

Logic:
For each user with reminders enabled:
- verify subscription tier
- find documents with expiration dates
- calculate days_until_expiration
- if days_until_expiration is 30, 14, 7, or 0:
  - send email if eligible/enabled
  - send SMS if eligible/enabled

Prevent duplicate reminders.

Create reminder_logs table:

reminder_logs
- id
- user_id
- document_id
- reminder_type
- days_before
- sent_at
- status
- provider_message_id
- error_message optional

Before sending:
Check reminder_logs to make sure the same reminder was not already sent for:
- user_id
- document_id
- reminder_type
- days_before
