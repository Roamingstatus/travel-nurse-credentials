Add a security monitoring and risk logging system to Credanta.

Do NOT rebuild the app.
Do NOT expose logs publicly.
Do NOT log passwords, API keys, document contents, or private file URLs.

Goal:
Detect and log suspicious activity such as:
- admin access attempts
- fake admin URL probing
- repeated failed logins
- unauthorized attempts to access user data
- attempts to access another user's documents
- share link abuse
- upload abuse
- repeated 500 errors
- suspicious request patterns

────────────────────────
DATABASE
────────────────────────

Create table:

security_events

Fields:
- id
- event_type
- severity
- user_id nullable
- email nullable
- ip_address
- user_agent
- route
- method
- request_metadata JSON
- created_at
- resolved boolean default false

Severity values:
- low
- medium
- high
- critical

Event types:
- admin_access_denied
- admin_probe_detected
- login_failure
- login_bruteforce_suspected
- unauthorized_data_access
- file_access_denied
- share_token_invalid
- share_token_abuse
- upload_rejected
- upload_abuse_detected
- rate_limit_triggered
- turnstile_failed
- server_error
- suspicious_request

────────────────────────
HELPER FUNCTION
────────────────────────

Create helper:

log_security_event(event_type, severity, request, user=None, metadata=None)

Rules:
- never crash the app if logging fails
- log server-side warning only
- strip sensitive data
- never log passwords
- never log document contents
- never log full file URLs
- never log API keys or tokens

────────────────────────
WHERE TO LOG
────────────────────────

Log admin_access_denied when:
- user tries admin route without permission

Log admin_probe_detected when:
- route contains suspicious admin paths:
  /admin
  /administrator
  /wp-admin
  /phpmyadmin
  /cpanel
  /server-status
  /.env
  /config
  /backup

Return 404, not 403, for obvious probes.

Log unauthorized_data_access when:
- user tries to fetch/edit/delete another user's document
- user tries to include another user's document in packet
- user tries to access another user's account data

Log file_access_denied when:
- private file access fails ownership check

Log share_token_invalid when:
- invalid/expired/revoked share token is requested

Log share_token_abuse when:
- too many invalid share token attempts from same IP

Log upload_rejected when:
- file type not allowed
- file too large
- corrupted file
- suspicious filename

Log rate_limit_triggered when:
- upload, login, feedback, waitlist, share link, or admin routes hit limits

Log turnstile_failed when:
- Cloudflare Turnstile verification fails

Log server_error when:
- global 500 handler catches exception

────────────────────────
BASIC DETECTION RULES
────────────────────────

Add simple pattern detection:

1. Admin probing:
If same IP hits 5 suspicious admin paths within 10 minutes:
log critical:
admin_probe_detected

2. Login brute force:
If same IP/email has 5 failed logins within 15 minutes:
log high:
login_bruteforce_suspected

3. Share token abuse:
If same IP has 10 invalid share token requests within 10 minutes:
log high:
share_token_abuse

4. Upload abuse:
If same user/IP has 10 rejected uploads within 15 minutes:
log medium/high:
upload_abuse_detected

5. Repeated 500s:
If same route produces 5 server errors within 10 minutes:
log high:
server_error_pattern

────────────────────────
PRIVACY / SECURITY
────────────────────────

Do not store:
- passwords
- auth tokens
- API keys
- full uploaded document text
- file contents
- private storage URLs
- full payment details

Mask sensitive values:
- tokens
- emails if needed
- long query strings

Use safe metadata only.

────────────────────────
UI SAFETY
────────────────────────

Never show security event dashboard to non-admins.

If non-admin tries:
- log admin_access_denied
- return 404 or 403
- do not reveal admin route location

────────────────────────
TESTING
────────────────────────

Add tests:

- denied admin access logs event
- /admin probe logs event and returns 404
- unauthorized document access logs event
- invalid share token logs event
- rejected upload logs event
- global 500 logs server_error
- non-admin cannot access security dashboard
- admin can view security dashboard
- sensitive fields are not logged

────────────────────────
IMPORTANT

This is a security monitoring layer, not a replacement for:
- authentication
- authorization
- Cloudflare
- rate limiting
- private storage
- secure share links

Keep implementation lightweight, reliable, and privacy-safe.
