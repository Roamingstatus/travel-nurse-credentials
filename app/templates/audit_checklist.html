Issues — click to expand
Critical Cloudflare Turnstile field name mismatch — blocked 100% of uploads ▶

Cloudflare injects a hidden field named cf-turnstile-response (hyphen). The FastAPI parameter was declared as cf_turnstile_response (underscore). FastAPI never found a match, so the token was always an empty string.

verify_turnstile("") → logged "missing response token from client" → returned False → every non-premium upload was rejected before touching storage or the database.

Files: app/main.py (upload route, ~line 835), app/security.py (verify_turnstile, line 535)

Fix: Read the token via await request.form() using the exact hyphenated key. Already written in the codebase — needs deployment.

token = str((await request.form()).get("cf-turnstile-response", "") or "")
High XHR follows redirects silently — upload errors look like success ▶

The upload form is submitted via XMLHttpRequest. When the server returns a 302 redirect (on both success and failure), the browser follows it automatically. xhr.status always lands on 200 — the JS handler can't tell the difference.

Result: on a failed upload the scan overlay shows "Saved!" and the user is navigated to /documents, where no new document appears. The flash error message appears briefly but the visual state is wrong.

Fix: Detect XHR via the X-Requested-With: XMLHttpRequest header and return {"ok": false, "error": "..."} with HTTP 400 so the overlay can display it directly.

Medium APP_ENV not set — app runs as "development" in production ▶
  • Session cookies are not Secure-only (https_only=False)
  • Admin route is the predictable path /admin-dev instead of a randomized secret path
  • Dev-only tier toggle (/toggle-tier) is active in production
  • Env validation logs warnings instead of blocking startup on missing critical vars

Fix: Add APP_ENV=production in Replit Secrets (lock icon → Secrets).

Medium HEIC files advertised but silently rejected ▶

The file picker's accept attribute lists .heic, so iPhone users can select their photos. The server's MIME allow-list does not include image/heic or image/heif, and there is no HEIC magic-byte entry — so every HEIC upload is rejected with "File type not permitted."

File: app/templates/upload.html line 24, app/security.py ALLOWED_MIME_TYPES

Fix A (quick): Remove .heic from the accept attribute.
Fix B (better): Add image/heic and image/heif to ALLOWED_MIME_TYPES and a HEIC magic-byte entry to _MAGIC_TABLE.

Critical (long-term) Uploaded files and database wiped on every deployment ▶

Replit deployment containers use an ephemeral filesystem. app/uploads/ (all documents) and app/data/app.db (all users, documents, sessions) are deleted on every new deployment.

This is not the cause of the current upload failure (uploads never reached storage due to the Turnstile bug), but it is a launch blocker for a live service.

Fix:

  • Files → migrate to Cloudflare R2, AWS S3, or Backblaze B2. Update app/storage.py to use that SDK.
  • Database → migrate to hosted PostgreSQL (Replit Postgres, Neon, Supabase). Update app/db.py connection string.
